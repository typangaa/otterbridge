# weir v0.4.0 — Remove the MCP server; pure CLI agent orchestrator

**Status:** planned · **Type:** BREAKING · **Version:** 0.3.0 → 0.4.0

## Motivation

weir already exposes its full orchestration surface through the CLI
(`weir chat`, `weir workflow run`, `weir backend …`, `weir workflow …`),
and that CLI is how the user actually drives it — via the `/weir` Claude Code
skill (Bash). The MCP server (`weir serve`) was a second, parallel interface
over the same engines. For this single-user, CLI-via-skill workflow the MCP
server earns nothing it doesn't already get from the CLI, while costing:

- a 559-line `src/server/` (rmcp tool handlers) with **zero** integration tests,
- the `rmcp` + `schemars` dependencies,
- a manual version-sync tax (the serverInfo version literal in `tools.rs` had
  to be hand-bumped — it shipped stale once already, fixed in v0.2.1),
- and a long-running-process story (hot-reload) that no CLI command needs.

Decision (user, 2026-06-16): **remove the MCP server and all its logic**,
rebrand weir to a single-binary **CLI agent orchestrator**, and take the deep
cut — also drop the now-dead hot-reload infrastructure.

## Scope

### Removed
- `src/server/` (mod.rs + tools.rs) — the MCP server and its 6 tools.
- `weir serve` subcommand (`Command::Serve` + dispatch).
- `rmcp`, `schemars` dependencies.
- Hot-reload `ConfigManager` (`src/config/manager.rs`): `ArcSwap` + `notify`
  file-watcher. It had a single caller — `weir serve`. All other commands
  already do a one-shot `Config::load`. → drop `arc-swap`, `notify` deps.
- `[server]` config section (`ServerConfig`, only field was a cosmetic `name`).
- `cli::serve::reload_signal` (dead operator helper about the server).

### Kept
- The entire CLI: `chat`, `workflow run/list/add/remove`, `backend
  list/test/add/remove`, `validate`, `status`, `version`, `schema`.
- The `/weir` skill (CLI-based) — the surviving interface.
- Resilience (`ResilientBackend`) + metrics + `weir status` (reads metrics from
  disk; CLI flush path unchanged).
- 3-layer validation (syntactic → semantic → resilience).
- All five engines and their unit tests (Step 1).
- The hard non-goals: no HTTP client, no HTTP server, no API keys.

### Hard constraint reaffirmed
weir opens no network socket and listens on no port. v0.3 removed the HTTP
client; v0.4 removes the (stdio) MCP server. weir is now exclusively a
short-lived CLI process that spawns local agent CLIs.
